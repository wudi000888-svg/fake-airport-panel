# v2.1.0 Release Notes

v2.1.0 是一次技术债清理版本：业务数据正式进入 SQLite-only 阶段，移除 v2.0.0/v2.0.1 过渡期保留的旧 JSON runtime fallback。

## 升级边界

| 项目 | 说明 |
| --- | --- |
| 支持来源 | v2.0.1 |
| 不支持来源 | v1.x JSON-only 运行数据直接升级 |
| 核心数据 | `data/panel/fake-ui.db`、`fake-ui.db-wal`、`fake-ui.db-shm` |
| 仍保留文件 | `auth.json`、登录提示文件、审计日志、订阅访问日志、Xray/Hysteria2 配置 |

如果你仍在 v1.x，请先升级到 v2.0.1 并确认 SQLite 数据库正常生成，再进入 v2.1.0。

## 主要变化

| 类型 | 内容 |
| --- | --- |
| 数据层 | 用户、套餐、订单、节点、注册审核、管理员资料、链接设置和链上支付统一使用 SQLite repository |
| 订阅 | 已有用户购买不同套餐时按“新套餐覆盖”处理；同套餐购买才按续费叠加 |
| 清理 | 删除 JSON 导入 SQLite、SQLite 回导 JSON 两个过渡脚本 |
| 部署 | 安装脚本直接初始化 SQLite，不再生成旧业务 JSON 后二次迁移 |
| 备份 | 内置备份只纳入 SQLite 数据库、认证文件和日志，不再收集旧业务 JSON |
| 文档 | README、运维、安全说明更新到 v2.1.0 SQLite-only 口径 |

## 上线前检查

```bash
python3 -m pytest -q
docker compose up -d --build panel
docker compose ps
```

生产机上线前请先备份完整 `data/`、Xray 配置和 Hysteria2 配置。香港真实业务机不要跳过备份和回滚包。
