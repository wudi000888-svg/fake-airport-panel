# fake-ui v2.1.1

v2.1.1 是 v2.1.0 SQLite-only 之后的安全加固版本，重点收紧内置管理面板的会话、Cookie、CSRF、登录尝试和响应头策略。此版本不改变部署架构，也不迁移业务数据。

## 主要更新

| 类型 | 内容 |
| --- | --- |
| Session | Session cookie 增加 `Secure`，继续使用 `HttpOnly` 和 `SameSite=Lax` |
| CSRF | 已登录 API 写操作必须携带 `X-CSRF-Token` |
| 兼容边界 | 旧版不含 CSRF 字段的 session 直接失效，升级后用户需要重新登录 |
| 登录防护 | 登录失败按 IP+用户名限速，并清理成功登录后的失败计数 |
| 审计 | 登录成功、登录失败、触发限速都会写入审计日志 |
| 安全头 | Python 面板响应增加 nosniff、X-Frame-Options、Referrer-Policy、Permissions-Policy；HTML 响应增加 CSP |
| Nginx | 默认模板补充 HSTS、CSP、X-Frame-Options、nosniff、Referrer-Policy 和 Permissions-Policy |
| 前端 | 登录和 session 响应会保存 CSRF Token，非 GET API 请求自动携带 `X-CSRF-Token` |

## 升级说明

| 项目 | 说明 |
| --- | --- |
| 支持来源 | v2.1.0 |
| 数据迁移 | 无数据库迁移 |
| 用户影响 | 已登录用户需要重新登录以获取带 CSRF 的新 session |
| 部署模式 | 继续支持 Docker Compose 和 systemd-native panel 模式 |

生产环境建议继续使用 HTTPS 反代，并保留备份、最小权限、日志审计和敏感文件隔离。仓库发布包不包含 `.env`、`data/`、数据库、证书私钥、真实域名、用户密码、钱包私钥或生产 RPC Key。
