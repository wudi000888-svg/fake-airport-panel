# fake-ui v2.2.0

v2.2.0 是从 v2.1.2 升级到商业运营后台的一次集中发布。它把管理员首页升级为可观察的运营仪表盘，同时补齐流量采样、部署安全、前端恢复、可信反代 IP 解析和备份恢复原子校验。

## Updates

- 管理首页新增指标卡、流量趋势图、套餐分布图、用户流量排行和节点流量排行。
- SQLite 新增 `traffic_samples` 表，Xray 与 Hysteria2 配额采集会写入历史样本。
- 新增 `/api/admin/metrics/*` 管理员指标接口，支持 overview、traffic、top users、nodes 和 plans。
- 管理员用户列表新增今日流量、剩余流量和配额进度条。
- 管理端流量趋势按查询窗口补齐空桶，非整点或按天查询不再错位。
- Xray/Hysteria2 指标暂时不可用时不再清零已有流量 baseline。
- 图表渲染失败时降级为空态，不影响整个后台页面。
- `/assets/*`、`/favicon.ico` 和 `/favicon.svg` 返回静态缓存头，API 与 HTML 继续 `no-store`。
- 默认运行镜像固定版本，避免 `latest` 漂移。
- 新增 `scripts/package-code.py`，发布包默认排除 `.env`、`data/`、`generated/`、`artifacts/`、`.demo-runtime/` 和 `.superpowers/`。
- 移除公开 `/api/password-reset/request` 旧入口，找回密码只保留邮箱验证码流程。
- 登录限速、登录审计和订阅访问限速只信任来自本机或私网反代的客户端 IP 转发头。
- 已登录启动时业务 API 短暂失败会显示重试/退出恢复页，不再渲染半初始化界面。
- 备份恢复先写入 staging 目录并校验 SQLite 完整性，通过后再替换运行文件；损坏数据库备份不会覆盖当前有效库。

## Upgrade Notes

从 v2.1.2 可直接滚动升级到 v2.2.0。升级不需要导入旧备份，数据库会自动创建新增的 `traffic_samples` 表和索引。

运行态 `.env`、`data/`、`generated/`、证书、用户数据、订阅 token、真实域名、用户密码和私钥必须继续保留在服务器，不应提交到 Git。
