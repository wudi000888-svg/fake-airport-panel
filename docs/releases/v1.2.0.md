# fake-ui v1.2.0

v1.2.0 主要补齐加密货币收款能力，并增强单机多出口代理编排在真实部署中的稳定性。这个版本仍然保持 receive-only 设计：系统只需要收款地址和公开查询端点，不需要导入钱包私钥或助记词。

## 主要更新

| 模块 | 更新 |
|---|---|
| 链上收款 | 新增 USDT、USDC、ETH、BNB、BTC 收款方式 |
| USD 计价 | 套餐和订单继续按美元计价，创建付款时锁定对应加密货币金额 |
| 管理员配置 | 管理员只需要填写收款地址，链、RPC/API、Token 合约和小数位使用内置默认值 |
| 用户付款 | 新增二维码付款、可选 TXID 兜底、EVM/BTC 自动验账和确认后自动开通订单 |
| RPC 稳定性 | 增加公共 RPC 备用端点、EVM 日志范围拆分、自适应扫描范围、按付款时间缩小扫描窗口，以及 BSC/ETH 最小安全扫描窗口 |
| 订单体验 | 用户订单区分未付款、需要 TXID、历史订单等状态 |
| 管理端清理 | 已取消和已完成订单不再显示客户侧付款操作 |
| Windows 部署 | 新增 Windows 安全的 Compose 部署脚本，减少 PowerShell/SSH 引号和 here-string 解析问题 |

## 安全说明

| 项目 | 说明 |
|---|---|
| 钱包私钥 / 助记词 | 不需要、不存储、不提交 |
| 运行数据 | `data/` 不进入仓库或源码包 |
| 证书 / token / 生产密钥 | 不进入仓库或源码包 |
| 收款方式 | 仅做 receive-only 收款和链上验账 |
