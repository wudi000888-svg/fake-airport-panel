# v2.1.0 SQLite-only Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove old business-data JSON runtime compatibility and make fake-ui v2.1.0 SQLite-only.

**Architecture:** Keep public store function names stable while replacing JSON fallback internals with SQLite repositories. Use `settings` for small singleton documents, dedicated tables for users/nodes/registrations, and DB-only backup semantics.

**Tech Stack:** Python 3.12 standard library, SQLite (`sqlite3`), existing pytest suite, Docker Compose preflight on Singapore VPS.
